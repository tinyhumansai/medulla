# Features

The [Overview](../README.md) explains *why* an orchestrator model exists. These
pages explain *what* it does day to day: the capabilities you configure, drive,
and see in the terminal app.

* [Memory](memory.md) — the persona pack distilled from your own coding-agent
  history, and the orchestrator's durable scratch space.
* [Workers and Sessions](workers-and-sessions.md) — what runs your work, how it
  is dispatched, and what survives a restart.
* [MEDULLA.md Workspace Profiles](workspace-profiles.md) — a short, authored file
  that tells the orchestrator what a repository *is*.
* [Orchestrator Routing](routing.md) — how planning, execution, and compression
  land on different models and harnesses.
* [Token Efficiency and Budgets](token-efficiency.md) — keeping the reasoning
  surface small, and spending a budget down to the last useful token.

These are product pages. For flags, file paths, and schemas, the
[Developers](../developers/README.md) section carries the operational detail, and
each page below links across where the two meet.
